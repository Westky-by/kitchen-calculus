## เพิ่มตัวกรองเจาะจง ปี/เดือน ใน Dashboard สรุป

### เป้าหมาย
ปรับ dropdown "ช่วงเวลา" ในหน้า Dashboard สรุป (`src/components/TaxInvoiceDashboard.tsx`) ให้เมื่อผู้ใช้เลือก preset บางค่า จะมีช่องเลือก **ปี (YYYY)** และ **เดือน (MM)** แบบเจาะจงเพิ่มขึ้นมา แทนการยึดกับ "ปัจจุบัน" อย่างเดียว

### พฤติกรรมใหม่ของแต่ละ preset

| Preset ที่เลือก | ช่องเสริมที่โผล่ขึ้นมา | ช่วงวันที่ที่คำนวณ |
|---|---|---|
| วันนี้ | — | วันนี้ |
| สัปดาห์นี้ | — | สัปดาห์นี้ (จันทร์–ปัจจุบัน) |
| เดือนที่ระบุ | เลือก **ปี** + **เดือน** | วันที่ 1 – วันสุดท้ายของเดือนนั้น |
| ปีที่ระบุ | เลือก **ปี** | 1 ม.ค. – 31 ธ.ค. ของปีนั้น |
| กำหนดเอง | ใช้ช่อง "จากวันที่/ถึงวันที่" เดิม | ตามที่กรอก |

- ค่าเริ่มต้นของ **ปี**/**เดือน** = ปี/เดือนปัจจุบัน
- รายการ **ปี** สร้างจากข้อมูลจริง (ปีที่พบใน `doc_date`) + ปีปัจจุบัน เรียงจากใหม่→เก่า
- รายการ **เดือน** = 01–12 พร้อมชื่อเดือนภาษาไทย (มกราคม–ธันวาคม)

### รายละเอียดทางเทคนิค

- แก้ไฟล์เดียว: `src/components/TaxInvoiceDashboard.tsx`
  - เปลี่ยน type `Preset` จาก `'month' | 'year'` เป็น `'month-pick' | 'year-pick'` (หรือคงชื่อเดิมแต่เพิ่ม state `pickYear`, `pickMonth`)
  - เพิ่ม state: `pickYear: number`, `pickMonth: number` (1–12)
  - แก้ `getRange()` / `useMemo(range)` ให้รับ `pickYear`, `pickMonth` มาคำนวณ from/to ของเดือน/ปีที่เลือก (วันสุดท้ายของเดือน = `new Date(y, m, 0).getDate()`)
  - Render conditional: ถ้า `preset === 'month'` โชว์ Select ปี + Select เดือน; ถ้า `preset === 'year'` โชว์ Select ปีอย่างเดียว
  - `compareCards` ("วันนี้/สัปดาห์นี้/ปีนี้") ยึด "ปัจจุบัน" เหมือนเดิม ไม่เกี่ยวกับตัวเลือก
- ไม่แตะ DB, ไม่แตะ tab อื่น, ไม่แตะการคำนวณ KPI/กราฟ/ตาราง (แค่ป้อน range ใหม่)
